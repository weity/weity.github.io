Markdown 学习

  • Markdown

突然发现语法上还是有些差异的。只能作参考了。
文章内容根据csdn的教程整理：

常规

斜体：用号或_包围 斜体 斜体
粗体：用号或者__包围 粗体 粗体
粗斜体：用包围 重点内容
划掉文字：用—包围 —删除—

标题：#号开头

一号 开头有一个#号

二号 开头有两个#号

引用：用>号。用多个>号进行嵌套引用。

引用内容,开头一个>号

嵌套引用，开头有两个>号。

代码格式化：
行内代码：用`号包围 var a;
多行代码：用三个`开始，然后指定语言，结束的时候用三个`号结束。

用代码演绎day day up

列表：

无序列表 用-号+空格 快捷键 ctrl+u

  • apple
  • banana
  • orange

有序列表 用数字+句点 快捷键 ctrl+o

  1. c
  2. java
  3. css

嵌套列表 用TAB将嵌套内容缩进

  1. fruit
    1. apple
    2. orange
    3. banana
  2. language
    • c
    • java
    • css

多段内容：缩进四个空格或者一个TAB。

  1. 列表
    段落一
    段落二
  2. 列表

链接

说明文本用中括号包围，url用小括号包围,在url要添加标题，用双引号引起。该标题能在鼠标移动到链接上时显示。

csdn Markdown 教程地址

参考式链接

先定义后调用

定义方式：中括号包围一个字符串（用于之后引用）+：+url+tilte（同样，tilte要用双引号引起来）

引用方式：
用中括号将文本包围，之后紧跟定义的字符串（该字符串同样用中括号包围）。
[Goole一下][Google]

链接图片

行内式格式：！+[altText]+(url+”title”)

参考式格式：

  1. 定义：[string1]+:+url+space+”altString”
  2. 引用：!+[title]+[string1]

目录 table of content

格式：[]+toc

[toc]目录

脚注

先定义后使用

swift[^name] is a beautiful girl[^like].

表格

在|中添加：可以修改对齐方式。

name sex age
jack boy 12
merry girl 11

[^name]:swift:come from America.

[^like]:like:beautiful girl.

公式编辑 基于MathJax->Latex

行内格式：用\$包围。
$s=\pi*r^2$

陈列公式：用两个\$\$包围。
$$
\Gamma(z)=\int_0^\infty t^{z-1} e^{-t} dt
$$
